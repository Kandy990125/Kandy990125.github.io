Active | Enable

  • Active：控制 GameObject 激活状态的；
  • Enable：控制 GameObject 里面的Component组件激活状态的。

Awake | OnEnable | Start

触发条件

  • Awake：GameObject Active；
  • OnEnable：GameObject Active；所在Script Enable；
  • Start：GameObject Active；所在Script Enable。

调用顺序

Awake -> OnEnable -> Start

调用时间

  • Awake：第一次 GameObject SetActive；
  • OnEnable：每次 GameObject Active 状态或者 Script Enable 状态变化时都会被调用；
  • Start：在 Update 第一次更新前被调用。

使用场景

  • Awake：初始化自身的属性；
  • Start：初始化依赖于别的组件的属性。

FixUpdate | Update | LateUpdate

【Unity Doc】事件函数的执行顺序

调用间隔

  • FixedUpdate：每隔固定时间调用一次；
  • Update：每帧调用一次；
  • LateUpdate：每帧调用一次；

调用顺序

Update / FixedUpdate -> LateUpdate

每帧的调用次数

  • FixedUpdate：调用 FixedUpdate 的频度常常超过 Update。如果帧率很低，可以每帧调用该函数多次；如果帧率很高，可能在帧之间完全不调用该函数；
  • Update：每帧调用一次；
  • LateUpdate：每帧调用一次；

使用场景

  • FixedUpdate：应用于物理碰撞和倒计时等需要精准数据的操作。
    • 如果把物理操作放在Update中会产生抖动。原因：碰撞检测的时间是固定的，当物体发生碰撞时，如果人物未停止移动，则下一帧物体会继续移动。
      （Update的更新频率太快大于了碰撞检测的频率，物体已经向前移动了，但还未进行碰撞检测，因此人物会穿越过去，待碰撞检测完毕后人物又会被弹出去。）
  • LateUpdate：跟随第三人称摄像机。如果在 Update 内让角色移动和转向，可以在 LateUpdate 中执行所有摄像机移动和旋转计算。这样可以确保角色在摄像机跟踪其位置之前已完全移动。
